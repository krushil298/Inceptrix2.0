## ğŸ”€ Git Branching Strategy

We follow **Git Flow**:

| Branch | Purpose |
|--------|---------|
| `main` | Production-ready code |
| `develop` | Integration branch for features |
| `feature/<name>` | New features (branch off `develop`) |
| `bugfix/<name>` | Bug fixes |
| `hotfix/<name>` | Urgent production fixes |
| `release/<version>` | Release preparation |

### Workflow
1. Create a feature branch: `git checkout -b feature/your-feature develop`
2. Commit your changes with clear messages
3. Push and open a Pull Request to `develop`
4. Get at least **1 code review** before merging
